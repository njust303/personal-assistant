## 二、文档管理规范

### 2.1 文档命名规范

**格式：** `类型 - 描述 - 版本。md`

**示例：**
- `PRD-个人理财助手-v1.0.md`
- `设计 - 架构设计-v1.0.md`
- `会议记录 -20260312-项目启动.md`

**版本号规则：**
- `v1.0` - 正式版
- `v1.1` - 小版本更新
- `v2.0` - 大版本更新

---

### 2.2 PRD 变更流程

```
1. 提出变更需求
   ↓
2. 评估变更影响
   ↓
3. 更新 PRD 文档
   ↓
4. 记录变更日志
   ↓
5. 通知相关人员
   ↓
6. 执行变更
```

**变更日志记录内容：**
- 变更日期
- 变更内容
- 变更原因
- 影响范围
- 变更人

---

### 2.3 文档审查周期

| 文档类型 | 审查频率 | 负责人 |
|---------|---------|--------|
| PRD | 每次变更前 | 项目负责人 |
| 设计文档 | 开发前审查 | 技术负责人 |
| 测试报告 | 每次测试后 | 测试负责人 |
| 会议纪要 | 每次会议后 | 会议记录人 |
